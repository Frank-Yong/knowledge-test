- File: `src/OperatingContextMcp.Server/Worker.cs`
- Added `TryWriteMessageAsync(Stream output, JsonObject response, CancellationToken ct)` helper that wraps protocol writes and catches:
  - `IOException`

## Why this matters
- Prevents unhandled exceptions when stdout is closed/disposed (e.g., broken pipe from MCP host/client disconnect).
- Allows worker loop to stop cleanly instead of crashing host unexpectedly.
- Preserves normal protocol behavior for healthy output stream writes.
