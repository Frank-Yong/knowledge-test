- File: `src/OperatingContextMcp.Server/Worker.cs`

  - to `FatalProtocolException("MCP frame is missing a valid Content-Length header.")`
- Effect: worker stops loop on malformed header rather than attempting to continue on potentially desynchronized stream state.

## Why this matters
- Avoids dropping potentially valid trailing bytes in ways that can permanently desync frame parsing.
- Makes malformed header handling deterministic and conservative under hostile/invalid input.
